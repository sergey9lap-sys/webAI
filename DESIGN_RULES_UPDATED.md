# DESIGN_RULES.md — Updated Additions

## Heading Size Rules

Do not create:
- absurdly huge headings;
- tiny corner labels that act as section titles;
- headings that visually disappear.

A section title must be clearly visible and feel important.

## Attention Hierarchy Rule

If a block is intended to communicate an idea through text,
the decorative elements must not steal attention from the text.

Before finishing a section ask:

What is the first thing the user notices?

If the answer is not the intended content,
the hierarchy is wrong.

## Card Alignment System

When multiple cards belong to one group:

Align across cards:
- numbers;
- badges;
- headings;
- subheadings;
- body text;
- dividers;
- avatars;
- buttons.

Cards must feel like one system.

## Service Blocks

If the goal of the block is to explain an approach,
avoid turning it into a catalog of cards.

Prefer:
- strong heading;
- supporting explanation;
- atmosphere;
- visual movement.

Not every concept should become a grid.
